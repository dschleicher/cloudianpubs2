Manage Users

Users & Groups --> Manage Users

In the CMC's Manage Users section you can add, edit, and delete HyperStore service users. Note that all users must belong to a user group -- therefore you must create one or more user groups before you can add any users.

Add a User

NOTE regarding LDAP authentication If you have your HyperStore system configured to use LDAP authentication, the system automatically provisions users when they log into the CMC with their LDAP credentials. Do not use the CMC's Add New User function if the system is configured to use LDAP authentication. Users that you add through the CMC's Add New User function will not be able to log into the CMC if the CMC is using LDAP authentication. (The exception is system admin users -- if you want to create another system admin user you can do that with the instructions below, and they will be able to log into the CMC). For more information on LDAP authentication see link:HyperStore-Admin-Guide.html#provisioning-ldap[Integrating with an LDAP System] in the _Cloudian HyperStore Administrator's Guide_.

  1. In the Manage Users page, click New User. This opens the Add New User panel.

  1. In the Add New User panel, complete the user information:
User ID
  • Must be unique within the group.
  • Only letters, numbers, dashes, and underscores are allowed.
  • Maximum allowed length is 64 characters.
  • The following user IDs are reserved for system use and are not available to individual users: "anonymous", "public", "null", "none", "admin", "0".
 User Type

From the drop-down list select the type of user that you want to create:

  • User -- Regular user of the storage service. He or she will be able to use the CMC to create storage buckets, upload and download objects, display reports on their service usage, and manage their service access credentials.
  • Group Admin -- Administer of a particular group of storage service users. He or she will be able to use the CMC to perform administrative functions for their group, such as adding users or setting user-level QoS profiles. A group admin has a storage service account and can upload their own objects to storage. A group admin can also manage stored objects on behalf of particular users within the group.
  • System Admin -- System administrator. He or she will be able to use the CMC to perform a variety of system administration and service administration tasks. System admins do not have a storage service account and cannot upload their own objects to storage. However, system admins can manage stored objects on behalf of particular service users.
 Group Name

From the drop-down list choose the group in which the new user will be created.

This field is hidden and not relevant if User Type is System Admin. All system admin users belong to the System Admin group.

 Password / Confirm Password

The user's password for logging into the CMC. Maximum allowed length is 64 characters.
  1. Optionally, click *More* to display additional fields:
Full Name

Optional. Maximum allowed length is 64 characters.

Address and contact information

Rating Plan
  • Rating plan to assign the user for billing calculation purposes. A new user's rating plan assignment defaults to whatever rating plan you've assigned to the group to which the user belongs. If you assign a user a different rating plan than the plan assigned to his or her group, the individual plan assignment will be applied to that user rather than the group default plan.
  • If you don't choose a rating plan for the user, the user is automatically assigned the default rating plan for the user's group.
  • If your service deployment has multiple service regions, rating plan assignment is on a per-region basis. A "Region" drop-down list displays which includes each of your region names. For each region, assign the user a rating plan that will apply to the user's service use within that region.
  1. Click Save.
 Retrieve a User Or a List of Users

In the Manage Users page you can retrieve a single user or a filtered list of users.

Select or enter your user list filtering criteria. You can filter by:

User ID Starts With
  • Specify a user ID prefix to retrieve a list of users whose IDs start with that prefix.
  • Specify a complete user ID if you want to retrieve just that one user.
  • Leave this field blank if you want to filter exclusively by the other criteria.

This field is case-sensitive.
Group Name

This is a drop-down list of group names in the system. This includes "System Admin" which is the group to which system administrators belong.
User Type

Select the type of user(s) to retrieve:

  • User — Regular service users.
  • Group Admin — Group admin users.
  • All — Regular service users, group admin users, and system admin users.
 User Status

Select the status of the user(s) to retrieve:

  • Active — Users who are currently allowed service access.
  • Inactive — Users whose service access credentials have been deactivated but who have not been deleted from the system. They may still have objects in storage.
  • All — Active users and inactive users.
