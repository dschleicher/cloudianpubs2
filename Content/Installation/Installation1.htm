  1. Choose one of your HyperStore hosts to be the Puppet master node, from which you will manage the installation and ongoing cluster configuration.

See more detail on this step.

The Puppet master node is the node on which you will run the HyperStore install script, which uses the open source version of Puppet to propagate initial configuration settings to the member nodes in your HyperStore cluster. This will also be the node from which you use Puppet for ongoing, centralized configuration management of your HyperStore cluster. The HyperStore system is designed to utilize Puppet for cluster configuration management, and the system is packaged and installed in a way that enables you to use Puppet from the outset, without doing any explicit set-up of Puppet. (For more information on using Puppet for ongoing cluster configuration management, see HyperStore Configuration in your Cloudian HyperStore Administrator’s Guide.)

HyperStore does not support using a stand-alone Puppet master node. The Puppet master node must be one of the nodes on which you will install and run HyperStore.
  1. On the Puppet master node, log in as root and create an installation staging directory (such as /home/CloudianPackages).
See more detail on this step.

On the node that you’ve selected to be your Puppet master node, log in as root and create a staging directory from which to run the HyperStore installation. Then change into the staging directory. For example:

root# mkdir /home/CloudianPackages
root# cd /home/CloudianPackages

For a HyperStore appliance install, the installation directory has already been created for you: /root/CloudianPackages. Change into that directory.

The staging directory on your Puppet master node must be kept intact throughout the life of your HyperStore service. The Cloudian Management Console will invoke scripts in this staging directory when you use the Console to perform certain system operations.
