hyperstore_data_directory

Quote-enclosed, comma-separated list of mount points to use for S3 object storage.

In a production environment, use dedicated disks for S3 object storage. Do not use the same disks that are storing the cassandra_commit_log_directory and cassandra_data_directory. For more information about HyperStore disk and mount point requirements, see File System.

The system will automatically assign virtual nodesThis is the description of a virtual node. (vNodes) to each of your S3 data mount points, in a manner that allocates an approximately equal total token range to each mount point. For more information about HyperStore vNodes see HyperStore vNodes. Note that the HyperStore system allocates an approximately equal total token range to each S3 data mount point even if the mount points are associated with disks of different sizes.

Do not change the hyperstore_data_directory setting once your clusterThis is the definition of a HyperStore "cluster". is operational.

Do not use symbolic links when specifying your mount points for the hyperstore_data_directory setting. The HyperStore system does not support symbolic links for these directories.

Example of a multiple mount point configuration = "/cloudian1,/cloudian2,/cloudian3,/cloudian4"

Default = Set during installation based on operator input, if host has multiple disks. For hosts with only one disk, default is /var/lib/cloudian

disk.write.threadpool.corepoolsize

"Core" size of each per-disk threadpool that the HyperStore Service uses to execute writes to disk, in support of S3 operations (such as when new S3 objects are written to disk). On each HyperStore Service node, there is one write threadpool for each HyperStore data disk on the node. When a write is to be done to a particular disk and there are no idle threads in the threadpool for that disk, then:

  • If fewer than this many threads are in the threadpool, the threadpool executor will create a new thread.
  • If this many or more threads are in the threadpool, the threadpool executor will queue the new task.

Default = 5
